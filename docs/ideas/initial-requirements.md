## プロジェクト概要

エクセル方眼紙形式で作成されたExcelファイル（.xlsx）を、Markdownドキュメントに変換するPython 3.12製CLIツール。

**重要な前提**: 通常のスプレッドシート変換（表→テーブル）ではなく、**文書構造**（見出し・段落・リスト）として認識・変換することが目的。

---

## エクセル方眼紙とは

日本のビジネス・行政文書で広く使われるExcelの使い方。

- セルを極小サイズ（方眼紙状）に設定し、ワープロ的に使う
- セルを結合してテキストブロックを作る（または結合せず隣の空セルにあふれるだけ）
- **列の位置**でインデント・階層を表現する
- **フォントの太字・サイズ**で見出しレベルを表現する
- **行の空き**でセクション区切りを表現する
- **背景色**でセクションや強調を表現する

通常のExcel読み込みライブラリでは「行=レコード、列=フィールド」として処理されるが、方眼紙ではこの前提が成り立たない。

「本物の方眼紙」はさらに雑然としている:
- セル結合をほとんど使わず、テキストが隣の空セルに視覚的にあふれるだけ
- 改行は「1行下のセルに書く」
- 横並び項目は「文字数分だけ列をスキップして次のセルに書く」

---

## 認識すべき文書構造

### 1. 見出し（Heading）

フォントサイズ・太字・列位置の組み合わせで判定する。

| 条件 | 見出しレベル |
|------|-------------|
| フォントサイズ >= 18pt | H1 |
| フォントサイズ >= 14pt かつ 太字 | H2 |
| フォントサイズ >= 12pt かつ 太字 | H3 |
| 太字 かつ 左端位置 かつ 広いスパン | H3 |
| 太字 かつ インデント0 | H4 |
| 太字 かつ インデント1 | H5 |
| 太字 かつ インデント2以上 | H6 |

### 2. 段落（Paragraph）

通常のテキストブロック。結合セル1単位をひとつの段落として扱う。

### 3. リスト項目（List Item）

- インデント位置（列の位置）から判定
- ネスト対応（インデント深さ = ネストレベル）
- `1.` や `1)` で始まる場合は番号付きリスト

### 4. 表（Table）

- グリッド状のセル配置（2行×2列以上）を検出
- GitHub Flavored Markdown（GFM）のテーブル形式に変換
- 最初の行をヘッダー行として扱う
- 表内でない領域には影響を与えない

### 5. 空行（Blank）

以下の条件で自動挿入する:
- ブロック間の行ギャップが大きい（方眼紙のグリッドピッチ × 2以上）
- 背景色が変わる（セクション境界）

## 技術要件

### 言語・実行環境

- Python 3.12
- 依存ライブラリ: `openpyxl >= 3.1.0`（.xlsx読み込み）
- 追加依存なし（pandasなどは不使用）

### CLIインターフェース

```bash
# 基本使用
python -m excel_to_markdown input.xlsx

# 出力先指定
python -m excel_to_markdown input.xlsx -o output.md

# シート指定
python -m excel_to_markdown input.xlsx -s "シート名"
python -m excel_to_markdown input.xlsx -s 0  # インデックス指定

# 調整オプション
python -m excel_to_markdown input.xlsx --base-font-size 10.5

# デバッグ出力（パース結果をJSONでstderrに出力）
python -m excel_to_markdown input.xlsx --debug
```

### テスト

- `pytest` によるユニットテスト・統合テスト
- openpyxl でプログラム的にテストフィクスチャ（.xlsx）を生成
- バイナリの.xlsxファイルはgoldenテスト用に最小限保持

## コアアルゴリズムの設計方針

### セル読み込み

- `openpyxl.load_workbook(path, data_only=True)` で読み込み
  - `data_only=True`: 数式ではなく計算済みの値を読む
- 結合セル（`ws.merged_cells.ranges`）の起点セルのみ処理
- 非起点セルはスキップ（値はNoneになる）

### インデントの計算

- 全テキストブロックの `left_col`（左端列番号）を収集
- 列幅の中央値（`col_unit`）を基準単位とする
- `col_unit * 1.5` 以内の列は同一インデントレベルにまとめる
  - 方眼紙では著者が列位置を完全に揃えないことがある→この許容幅で吸収

### 表の検出

- 全テキストブロックを走査し、矩形グリッドを形成するものを検出
- 検出条件: 2行以上 × 2列以上、各行の列境界が一致
- 保守的な検出（曖昧なケースは通常の段落として残す）

---

## 非機能要件

- 大きなExcelファイル（1000行以上の方眼紙）でも実用的な速度で動作すること
- 変換結果のMarkdownが人間に読みやすいこと（余分な空行・空白を除去）
- エラーメッセージが分かりやすいこと（ファイルが存在しない、シートが見つからない等）

## 追加仕様（確認済み）

### 同一行に複数テキストブロックが横並びの場合（ラベル:値パターン）

方眼紙でよく見られる「ラベルと値が横に並ぶ」パターンを認識して変換する。

**判定条件:**
- 同一行に2つのテキストブロックが横並びに存在
- 左のブロックが短いテキスト（ラベル候補）
- 右のブロックが値

**変換結果:**
```markdown
**氏名:** 山田太郎
**所属:** 開発部
```

3列以上横並びの場合は左から順にスペース区切りで1段落に結合する。

---

### セル内改行（Alt+Enter）の扱い

Excelのセル内改行（`\n`）はMarkdownのハードブレークとして変換する。

```markdown
1行目のテキスト
2行目のテキスト
```

（行末に半角スペース2つ + 改行）

---

### 複数シートの扱い

デフォルトで全シートを1つのMarkdownファイルに統合する。
各シートはシート名をH1見出しとして区切る。

```markdown
# Sheet1

...シート1の内容...

---

# Sheet2

...シート2の内容...
```

`-s シート名` または `-s インデックス` で特定シートのみの変換も可能。

---

### 配布・実行方法

`python -m` での実行を想定。パッケージ配布（pip install）はスコープ外。

```bash
git clone <repo>
pip install -e .
python -m excel_to_markdown input.xlsx
```

---

### 印刷領域の扱い

Excelに印刷領域が設定されている場合は**印刷領域内のみ**を変換対象にする。
印刷領域外（作業メモ・下書き等）は無視する。

- 印刷領域が未設定の場合はシート全体を対象にする
- 非表示の行・列は変換対象から除外する

---

### 文字書式の変換

太字以外の主要な文字書式もMarkdownに変換する。

| Excel書式 | Markdown変換 |
|-----------|-------------|
| 太字 | `**text**` |
| イタリック | `*text*` |
| 取り消し線 | `~~text~~` |
| 下線 | `<u>text</u>`（HTMLタグ） |

文字色・背景色はMarkdown変換せず、構造判定（セクション境界等）にのみ使用する。

---

### 変換方針（ベストエフォート）

構造を完全に認識できない場合（フォントサイズ未設定・曖昧なレイアウト等）も、セルの内容を必ず段落として出力する。情報を捨てない。

```markdown
<!-- WARNING: 構造を認識できませんでした -->
セルの内容をそのまま段落として出力
```

認識できなかった要素はWARNINGコメントをMarkdownに挿入し、`--debug` オプション時は詳細をstderrにも出力する。

---

### セルコメント・メモの扱い

Excelのセルコメント（メモ）はMarkdown脚注として変換する。

```markdown
本文テキスト[^1]

[^1]: セルコメントの内容
```
