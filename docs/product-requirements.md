# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**excel-to-markdown** - エクセル方眼紙 → Markdown コンバーター

### プロダクトコンセプト
- **文書構造の自動認識**: セルの位置・フォント書式・結合情報から、見出し・段落・リスト・表を自動判定する
- **情報を捨てないベストエフォート変換**: 構造が認識できない部分も段落として必ず出力し、WARNINGコメントで通知する
- **シンプルなCLI**: 追加学習不要で `python -m excel_to_markdown input.xlsx` の1コマンドで変換できる

### プロダクトビジョン

日本のシステム開発現場では、要件定義書・設計書・仕様書がExcel方眼紙として大量に蓄積されている。AIを活用したコード生成・レビュー・分析を進めようとしても、これらの文書をそのままAIに入力できず、「過去の設計資産が活かせない」という問題が生じている。

本ツールは、Excel方眼紙の視覚的な構造（フォント・セル位置・結合）からMarkdownを自動生成することで、眠っていた設計資産をAIが扱えるテキストに変換する。変換精度は完璧でなくてもよい——情報が失われないベストエフォート変換を原則とし、AIへの入力や人間の確認・修正がしやすい出力を提供することを優先する。

### 目的
- 過去の要件定義書・設計書・仕様書（Excel方眼紙）をAIが読めるMarkdownに変換し、AI活用の障壁を取り除く
- 構造の自動認識により、手動でのMarkdown書き起こし作業を削減する
- 既存の方眼紙資産をテキストとして検索・バージョン管理・再利用できるようにする

- **典型的なワークフロー**: 顧客からExcel設計書を受領 → CLIで変換 → 目視確認・軽微な修正 → リポジトリにコミット → AIに設計書を読ませてコード生成・レビューを依頼

- 社内の業務マニュアル・申請書がすべてExcel方眼紙で、AI検索・AI要約ができず困っている
- RAGシステムへの取り込みのためにMarkdown化したい
- コマンドライン操作に慣れており、バッチ処理に組み込んで使いたい

## 成功指標（KPI）

### プライマリーKPI

- **テキスト保全率**: 変換後Markdownに、入力Excelの全テキスト内容が含まれる率 **100%**
  - ベストエフォート方針により、構造が不明でも段落として必ず出力することで達成
  - 測定: 代表的な方眼紙10種に対してテキスト抽出比較
- **構造認識精度**: 代表的な方眼紙文書における、見出し・段落・リスト・表の分類正解率 **85%以上**
  - 測定: 以下の6パターンのテストフィクスチャに対する自動テストで一致率を計測（複雑な組み合わせは追加可）
    - シンプルな見出し+段落構造
    - ネストリスト（3段階以上）
    - ラベル:値パターン（2列・3列以上）
    - 見出し・表・段落が混在する文書
    - セルコメント付き文書
    - 印刷領域設定あり/なしの文書
- **変換速度**: A4縦1ページ相当（約100行×50列）の方眼紙を **3秒以内** に変換完了
  - 測定: pytestの統合テストで計測

### セカンダリーKPI

- **エラー率**: 正常な.xlsxファイルで例外が発生する率 **0%**
  - 「正常」の定義: パスワード保護なし・openpyxlで開けるファイル
  - 測定: テストスイート全パス
- **ユニットテストカバレッジ**: **80%以上**

## 機能要件

### コア機能（MVP）

#### 1. Excelファイル読み込み

**ユーザーストーリー**:
ドキュメント管理担当として、方眼紙Excelを変換するために、.xlsxファイルをCLIで指定して読み込めるようにしたい

**受け入れ条件**:
- [ ] `.xlsx` ファイルをパスで指定して読み込める
- [ ] 印刷領域が設定されている場合は印刷領域内のセルのみを変換対象とする
- [ ] 印刷領域が未設定の場合はシート全体を変換対象とする
- [ ] 非表示の行・列を変換対象から除外する
- [ ] 結合セルを正しく認識し、起点セルの値のみ処理する
- [ ] 存在しないファイルを指定した場合、わかりやすいエラーメッセージを表示する

**優先度**: P0（必須）

---

#### 2. 見出し検出

**ユーザーストーリー**:
ドキュメント管理担当として、方眼紙の視覚的な見出し構造をMarkdownに再現するために、フォント書式と位置から自動的に見出しレベルを判定してほしい

**受け入れ条件**:
- [ ] 以下の条件を上から順に評価し、最初に一致したルールを適用する（優先順位順）:
  1. フォントサイズ18pt以上 → H1
  2. フォントサイズ14pt以上かつ太字 → H2
  3. フォントサイズ12pt以上かつ太字 → H3（インデントレベル不問）
  4. 太字かつインデントレベル0 → H4（上記フォントサイズ条件を満たさない場合）
  5. 太字かつインデントレベル1 → H5
  6. 太字かつインデントレベル2以上 → H6

**優先度**: P0（必須）

---

#### 3. インデントと段落・リスト変換

**ユーザーストーリー**:
ドキュメント管理担当として、方眼紙の列位置で表現された階層構造を、Markdownのインデントリストで再現してほしい

**受け入れ条件**:
- [ ] セルの左端列位置からインデントレベルを自動計算する（`col_unit` = シート全体の列幅の中央値を基準単位とする）
- [ ] 列位置が `col_unit × 1.5` 以内のセルを同一インデントレベルとしてグループ化する（方眼紙での位置ズレを吸収）
- [ ] インデントレベル1以上かつ見出し条件を満たさないセルをリスト項目として変換する（`- text`）
- [ ] インデントレベル2以上のリスト項目はネストして変換する（`  - text`）
- [ ] `1.` や `1)` で始まるテキストは番号付きリストとして変換する
- [ ] インデントレベル0で見出し条件を満たさないセルは段落として変換する

**優先度**: P0（必須）

---

#### 4. 表（テーブル）検出

**ユーザーストーリー**:
ドキュメント管理担当として、方眼紙内のグリッド状の表をGFMテーブルに変換してほしい

**受け入れ条件**:
- [ ] 2行×2列以上の矩形グリッドを形成するセルブロックを表として検出する
- [ ] 検出した表をGitHub Flavored Markdownのテーブル形式（`| col | col |`）に変換する
- [ ] 最初の行をヘッダー行（`| --- | --- |`）として扱う
- [ ] 表の外側のセルには影響を与えない
- [ ] 曖昧なケース（グリッドが不完全）は表として検出せず段落として残す

**優先度**: P0（必須）

---

#### 5. ラベル:値パターン認識

**ユーザーストーリー**:
ドキュメント管理担当として、「氏名:　山田太郎」のような横並びのラベルと値を、読みやすいMarkdownに変換してほしい

**受け入れ条件**:
- [ ] 同一行に2つのテキストブロックが横並びで、左のテキストが20文字以下の場合にラベル:値パターンと判定し、`**ラベル:** 値` の形式に変換する
- [ ] 3列以上横並びの場合は左から順にスペース区切りで1段落に結合する（ラベル:値判定は適用しない）

**優先度**: P0（必須）

---

#### 6. 複数シートの統合変換

**ユーザーストーリー**:
ドキュメント管理担当として、複数シートのExcelを1つのMarkdownに変換してほしい

**受け入れ条件**:
- [ ] デフォルトで全シートを1つのMarkdownファイルに統合する
- [ ] 各シートはシート名をH1見出し（`# シート名`）として区切る
- [ ] シート間に水平線（`---`）を挿入する
- [ ] `-s シート名` または `-s インデックス` で特定のシートのみ変換できる

**優先度**: P0（必須）

---

#### 7. 文字書式の変換

**ユーザーストーリー**:
ドキュメント管理担当として、太字・イタリック・取り消し線などの文字装飾をMarkdownに引き継いでほしい

**受け入れ条件**:
- [ ] 太字セル（見出し以外）のインライン太字部分を `**text**` に変換する
- [ ] イタリック体を `*text*` に変換する
- [ ] 取り消し線を `~~text~~` に変換する
- [ ] 下線を `<u>text</u>`（HTMLタグ）に変換する
- [ ] セル内で部分的に書式が適用されている場合（リッチテキスト）もインライン変換する

**優先度**: P0（必須）

---

#### 8. セル内改行の変換

**受け入れ条件**:
- [ ] Excelセル内のAlt+Enter改行（`\n`）をMarkdownのハードブレーク（行末スペース2つ + 改行）に変換する

**優先度**: P0（必須）

---

#### 9. セルコメント・メモの脚注変換

**受け入れ条件**:
- [ ] Excelのセルコメント（メモ）をMarkdown脚注（`[^N]` / `[^N]: 内容`）として出力する
- [ ] 脚注番号は文書内で連番になる

**優先度**: P0（必須）

---

#### 10. 空行・セクション区切りの自動挿入

**受け入れ条件**:
- [ ] ブロック間の行ギャップが方眼紙のグリッドピッチ × 2以上の場合に空行を挿入する
- [ ] 背景色が変わる箇所でもセクション境界とみなして空行を挿入する（白=FFFFFF から非白、または前行と背景色RGBが異なる場合を「変わった」と判定する）
- [ ] 出力Markdownで3行以上の連続空行は2行に圧縮する

**優先度**: P0（必須）

---

#### 11. ベストエフォート変換（WARNINGコメント）

**受け入れ条件**:
- [ ] 構造を認識できないセルも段落として必ず出力する（情報を捨てない）
- [ ] 認識が不確かな要素にはMarkdownコメント（`<!-- WARNING: ... -->`）を挿入する

**優先度**: P0（必須）

---

#### 12. CLIインターフェース

**ユーザーストーリー**:
エンジニアとして、シェルスクリプトに組み込んで自動変換するために、CLIで柔軟なオプションを指定したい

**受け入れ条件**:
- [ ] 以下のコマンド形式で動作する:
  ```bash
  python -m excel_to_markdown input.xlsx              # 基本
  python -m excel_to_markdown input.xlsx -o out.md    # 出力先指定
  python -m excel_to_markdown input.xlsx -s "Sheet1"  # シート指定
  python -m excel_to_markdown input.xlsx -s 0         # インデックス指定
  python -m excel_to_markdown input.xlsx --debug      # デバッグ出力
  python -m excel_to_markdown input.xlsx --base-font-size 10.5
  ```
- [ ] `-o` 未指定時は入力ファイルと同じディレクトリ・同名の `.md` ファイルに出力する
- [ ] `--base-font-size` を指定すると、見出し判定のフォントサイズ閾値（18pt/14pt/12pt）をデフォルトベース（11pt）に対する倍率で再計算する（例: `--base-font-size 10.5` の場合、18pt閾値は `18 × 10.5/11 ≒ 17.2pt` に調整）
- [ ] `--debug` 指定時はパース結果（TextBlockリスト）をJSON形式でstderrに出力する
- [ ] `--version` でバージョンを表示する
- [ ] `--help` で使い方を表示する

**優先度**: P0（必須）

## 非機能要件

### パフォーマンス
- A4縦1ページ相当（約100行×50列）の方眼紙を **3秒以内** に変換完了（平均的なPC環境で）
- 1,000行以上の方眼紙でも実用的な速度（30秒以内）で変換完了（測定: 1,000行×50列のテストフィクスチャをpytestで計測）

### ユーザビリティ
- エラーメッセージは原因と対処法を日本語で表示する（例: 「ファイルが見つかりません: input.xlsx」）
- `--help` を見れば5分以内に基本操作を把握できる

### 信頼性
- 正常な.xlsxファイルで例外（クラッシュ）が発生する率 **0%**
- 変換中にメモリエラーが発生した場合、部分的な出力ファイルを残さず終了する

### 保守性
- ユニットテストカバレッジ **80%以上**
- 各モジュールの責務を明確に分離し、見出し判定ロジック等を独立してテスト可能にする

### 互換性
- Python 3.12 で動作
- OS: Linux / macOS / Windows（クロスプラットフォーム）

## スコープ外

明示的にスコープ外とする項目:
- マクロ・VBAの処理・実行
- 画像・図形・グラフの変換
- Excelのハイパーリンクの変換（Post-MVPとして将来検討）
- Google Sheets対応
- GUIインターフェース
- `pip install` によるパッケージ公開（リポジトリをクローンして使用）
- 縦書きセルの変換（横書きとして処理するか、段落として出力）
