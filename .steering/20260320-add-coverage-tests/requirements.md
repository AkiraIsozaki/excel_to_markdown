# 要求内容

## 概要

カバレッジが80%未満のモジュールにテストを追加し、プロジェクト全体のカバレッジを80%以上に引き上げる。

## 背景

PRDのセカンダリーKPI「ユニットテストカバレッジ80%以上」を達成するため。
現状のカバレッジ:
- `cli.py`: 0%（120行すべて未テスト）
- `merge_resolver.py`: 50%（`to_inline_runs()`関数が未テスト）
- `__main__.py`: 0%（3行、entrypoint）
- 全体: 72%

## 実装対象の機能

### 1. test_cli.py の新規作成

`cli.py` の全関数をテストする。

### 2. merge_resolver.py の `to_inline_runs()` テスト追加

`test_merge_resolver.py` に `to_inline_runs()` のテストケースを追加する。

## 受け入れ条件

- [ ] pytest 実行時に全テストがパスすること
- [ ] `cli.py` のカバレッジが80%以上になること
- [ ] `merge_resolver.py` のカバレッジが80%以上になること
- [ ] プロジェクト全体のカバレッジが80%以上になること

## スコープ外

- `__main__.py`（entrypointのみ、カバレッジ向上の費用対効果が低い）
- 新機能の追加
- リファクタリング
