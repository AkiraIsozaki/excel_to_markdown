# 要求内容

## 概要

PRDに定義されている未実装機能をすべて実装する。P1（.xls対応）、P2（ハイパーリンク変換・バッチ変換）、および非機能要件（1000行パフォーマンステスト・メモリエラー時の出力ファイル削除）を含む。

## 実装対象の機能

### 1. .xls 旧形式対応（P1）
- `reader/xls_reader.py` を新規作成し、xlrd 2.x でシートから `list[RawCell]` を抽出する
- `cli.py` の `_open_workbook` を `.xls` / `.xlsx` で分岐させる
- `.xls` ファイルを同じ変換パイプラインに通せること

### 2. メモリエラー時の出力ファイル削除
- `cli.py` の `_write_output` をアトミック書き込みに変更する（tmpファイル→rename）
- 書き込み失敗時に部分出力ファイルが残らないこと

### 3. 1,000行パフォーマンステスト
- `tests/test_integration.py` に `test_performance_1000rows` を追加する
- 1,000行×50列の方眼紙を 30 秒以内に変換できること

### 4. ハイパーリンク変換（P2）
- `RawCell` / `TextBlock` に `hyperlink` フィールドを追加する
- `xlsx_reader.py` でセルのハイパーリンクURLを抽出する
- `markdown_renderer.py` で `[text](url)` 形式に変換する

### 5. バッチ変換（P2）
- `cli.py` にディレクトリ入力モードを追加する
- ディレクトリを指定すると配下の全 `.xlsx`（`.xls` も含む）を一括変換する
- CLI: `python -m excel_to_markdown /path/to/dir/`

## 受け入れ条件

- [ ] `.xls` ファイルを CLI で指定して変換できる
- [ ] `.xls` 変換時にセル値・結合・フォント書式が正しく処理される
- [ ] 書き込みエラー時に部分出力ファイルが残らない
- [ ] 1,000行×50列テストが 30 秒以内に完了する（pytest でアサーション）
- [ ] ハイパーリンクを持つセルが `[text](url)` 形式に変換される
- [ ] ディレクトリ指定で配下の全 xlsx/xls を一括変換できる
