# 要求内容

## 概要

FastAPI + HTML5 ドラッグ＆ドロップによるブラウザ型Web UIを追加する。ユーザーは複数のExcelファイルをD&Dで投入すると、一括変換してMarkdownファイル一式をZIPでダウンロードできる。

## 背景

現在のCLIインターフェースはコマンドライン操作に慣れたエンジニア向けに設計されているが、より広いユーザー層（非エンジニア・Excelユーザー）が手軽に変換できる手段が求められている。ブラウザで動作するWebアプリとして実装することで、OS・環境を問わず利用できる。

## 実装対象の機能

### 1. Web UIサーバー（FastAPI + uvicorn）

- `python -m excel_to_markdown serve` または `python -m excel_to_markdown --serve` でローカルWebサーバーを起動する
- デフォルトポートは `8000`（`--port` オプションで変更可能）
- ブラウザが自動で開く（`--no-browser` オプションで無効化可能）
- Ctrl+C で終了

### 2. ドラッグ＆ドロップ変換UI（HTML5 + JavaScript）

- ファイルを画面にD&Dするか、クリックでファイル選択ダイアログを開ける
- 複数ファイル（.xlsx / .xls）を同時に投入できる
- 変換中はプログレス表示（スピナーまたはファイル名リスト）
- 変換完了後、ZIPファイルをダウンロードする
  - 単一ファイルの場合は .md ファイル直接ダウンロード（ZIPなし）も検討
- エラーファイルは個別にエラーメッセージ表示し、成功分はダウンロード可能

### 3. REST APIエンドポイント

- `POST /api/convert` : multipart/form-data でファイル受信、変換結果を返す
  - 単一ファイル → `text/markdown` でレスポンス
  - 複数ファイル → `application/zip` でレスポンス
- `GET /health` : ヘルスチェック（JSON）

## 受け入れ条件

### Web UIサーバー

- [ ] `python -m excel_to_markdown serve` でサーバーが起動する
- [ ] `--port 9000` でポート変更できる
- [ ] `http://localhost:{port}` にブラウザでアクセスできる
- [ ] Ctrl+C でサーバーが正常に停止する

### D&D UI

- [ ] .xlsx / .xls ファイルをD&Dエリアにドロップできる
- [ ] 複数ファイルを同時にD&Dできる
- [ ] クリックでファイル選択ダイアログが開く
- [ ] 変換中にスピナーまたは処理中インジケータが表示される
- [ ] 単一ファイル変換で .md がダウンロードされる
- [ ] 複数ファイル変換で ZIP がダウンロードされる
- [ ] 非対応拡張子ファイルをD&Dした場合、エラーメッセージが表示される
- [ ] 変換エラーが発生したファイルはエラー表示し、他ファイルの変換は継続する

### REST API

- [ ] `POST /api/convert` に .xlsx をアップロードすると Markdown テキストが返る
- [ ] `POST /api/convert` に複数ファイルをアップロードすると ZIP が返る
- [ ] `GET /health` が `{"status": "ok"}` を返す
- [ ] 非対応形式のファイルを送信すると 400 エラーが返る
- [ ] ファイルサイズ上限（50MB）を超えると 413 エラーが返る

## 成功指標

- ブラウザでアクセスしてD&Dから変換完了まで、コマンドライン操作が不要
- 既存の変換パイプライン（CLI）への影響なし（既存テストが全パスのまま）
- A4縦1ページ相当のExcelが5秒以内に変換・ダウンロード開始できる

## スコープ外

以下はこのフェーズでは実装しません:

- 認証・認可機能（ローカル利用前提）
- ファイルの永続保存（サーバー側に変換結果を保管しない）
- 変換オプション（--base-font-size 等）のUI設定
- 変換履歴の表示
- リモートデプロイ・Docker化
- WebSocket によるリアルタイム進捗通知
